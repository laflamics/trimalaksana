SYNC FLOW (Now Same as Packaging):

Device-to-Server:
  1. User creates/updates delivery note
  2. storageService.set() saves to localStorage immediately
  3. Dispatches 'app-storage-changed' event
  4. Debounced sync to server (5 seconds)
  5. Retries up to 3 times with exponential backoff

Server-to-Device:
  1. Component calls storageService.get()
  2. Returns local data immediately (non-blocking)
  3. Triggers background sync if data >30 seconds old
  4. Fetches from server via WebSocket
  5. Merges server data with local data
  6. Updates localStorage
  7. Dispatches 'app-storage-changed' event
  8. Component re-renders with new data

Real-Time Sync:
  1. Listens for 'app-storage-changed' events
  2. Auto-reloads when data changes on other devices
  3. No polling needed
